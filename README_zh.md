# Hermes V2 微内核 (Microkernel)

[English](README.md) | [中文版](README_zh.md)

Hermes V2 微内核 (Microkernel) 是为 Hermes Agent 设计的轻量级、安全且极具韧性的基础架构。

### 🎯 核心目标：极大地节省 Token 消耗
在传统的 AI Agent 运行模式下，每次对话都需要把所有的工具（Tools）说明、大量预设技能（Skills）和长篇的系统提示词（System Prompts）发送给大模型。这不仅会导致上下文变得非常臃肿，还会消耗极其巨大的 Token 数量，造成严重的成本浪费和响应延迟。

**本项目是如何解决这个问题的？（小白也能看懂的原理）**

1. **按需加载工具 (MCP - Model Context Protocol)**：
   我们不再把所有工具的说明一次性塞给 AI。微内核作为“调度中心”，只有当 AI 明确需要执行某项特定任务时，才通过 MCP 协议动态加载对应的工具说明和能力。用完即抛，绝不占用常驻上下文。

2. **把“技能”存入 MemPalace (记忆宫殿)**：
   原来 AI 需要记住的复杂长篇“操作手册”和“业务流程”（Skills），现在被剥离出来，存放在专门的知识库（MemPalace）中。只有当 AI 在处理特定场景时，微内核才会从记忆宫殿中精准检索出对应的一小段核心技能提供给 AI。这让主对话框里的上下文保持极度轻量（Lean Context）。

通过这套“按需加载 + 记忆外包”的机制，微内核有效避免了无效上下文的堆积，从而**极大地降低了每次对话的 Token 消耗**。

此外，微内核还提供强大的环境防护、全局自愈能力以及优化的配置管理，确保智能体运行环境的绝对稳定。

## 核心特性

- **安全的 MCP 桥接**: 为 MCP 工具提供安全隔离的桥接脚本，通过严格的环境变量解析防止任意代码执行和凭证泄露。
- **防污染 Shell 拦截器**: 在 Shell 配置文件（`.bashrc`、`.zshrc`）中注入防护层，支持多行深度检测，防止 YAML 解析逃逸导致的配置固化。
- **全局自愈插件**: 自动修复损坏的配置文件，并具备严格的类型检查机制，防止系统崩溃。
- **微内核技能组**: 内置必要的子智能体协同和系统维护技能，实现无缝的自治操作。
- **跨平台兼容**: 使用 POSIX 标准规范，全面兼容 Linux、macOS 和 BSD 操作系统。

## 安装指南

运行安装脚本以部署微内核架构：

```bash
chmod +x install.sh
./install.sh
```

## 目录结构
- `install.sh`: 核心自动化部署脚本。
- `plugins/global_auto_heal/`: 提供配置韧性的全局自愈插件。
